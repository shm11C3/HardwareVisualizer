# Security Policy

## Supported Versions
We provide security fixes for the latest stable release only.

| Version | Supported |
| ------: | :-------: |
| Latest  | ✅        |
| Older   | ❌        |

## Reporting a Vulnerability
**Do not open a public GitHub Issue for security vulnerabilities.**

Preferred: Use GitHub "Report a vulnerability" (Private Vulnerability Reporting / Security Advisories).

Alternative: If you cannot use GitHub reporting, contact the maintainer privately:
- Email: <m11c3.sh@gmail.com>

Please include:
- Affected version(s)
- Reproduction steps / PoC (if available)
- Impact assessment
- Any logs or screenshots that help

## Disclosure Policy
- We will acknowledge receipt within **7 days**.
- We will work on a fix and coordinate disclosure timing with the reporter.
- After a fix is released, we may publish a security advisory and/or release notes.
